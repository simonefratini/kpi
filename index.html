project
  • Actual numbers is referred to all open bugs from start of the project.
  • Average days to close is computated on all bugs closed in the month, independently when they was opened.
  • Drill-down on doughnut slices works only if project is set
    • Doughnut are a snapshot at certain moment. When you click on a slice, the number of bugs on filter overview of redmine page, it could be different.
team
  • Average days a bug is in charge on the current month is referred to today, so could be smaller that you aspect from previous months, especially at the beginning of the month.
  • In the same month, one bug could be managed by a team at different intervals, these are united.
  • One bug could be handled by more than a team, so the sum of different teams' bugs has no mean.
  • For "All Teams - One Company", bugs in the month could be only closed, moved has no mean.
data congruence
  • Teams must be mutually exclusive for members
  • For every project these fields must be the same value
    • Current Number: Open bugs (total in doughnut)
    • Current Number: Priority (total in doughnut)
    • Summarized Bugs Performance: Currently Open Bugs Absolute
  • For project "All" and team "All Teams - One Company"
    • these two fields must be the same value
      • Summarized Bugs Performance: Absolute Currently Open
      • Summarized Teams Last 12 Month Performance: Currently Owned
    • these two fields must be the same value
      • Summarized Bugs Performance: Last 12 Month Closed
      • Summarized Teams Last 12 Month Performance: Bugs Moved or Closed
    • the relation and the field must be same value
      • Summarized Bugs Performance: Last 12 Months Total plus difference between Absolute and Last 12 Months Currently Open
      • Summarized Teams Last 12 Month Performance: Total Bug Owned
  • Data are extracted from Monday to Friday, every hour from 8 AM to 8 PM
filter
  • Priority ≥ High, for every bugs is used the actual priority value
  • Almost average is sample average
    • Are not sample, when filter is high and project or team is set
    • Standard Error is sample standard deviation, useful because is not a normal distribution with constant variance for sample months
